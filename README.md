
# DevTools Demo — учебный сайт для SEO и разработчиков
[DevTools Demo](https://shutnik8.github.io/DevTools-Demo/)

Небольшой демо‑проект: одна HTML‑страница с намеренно допущенными ошибками и спорными решениями, чтобы тренироваться работать с DevTools в браузере (Console, Elements, Network) и делать обучающие скриншоты.

Проект подойдёт:
- начинающим SEO‑специалистам, которые хотят лучше понимать техническую сторону;
- разработчикам, которые помогают SEO/маркетингу и хотят видеть сайт глазами браузера и поискового бота;
- авторам обучающих статей и курсов — как источник простых, воспроизводимых примеров.

## Что демонстрирует страница

В коде специально заложены ситуации, которые часто встречаются на реальных проектах:

1. **Ошибка JavaScript в консоли**
   - В `<script>` вызывается несуществующая функция:
     ```js
     console.log('Инициализация демо-сайта для DevTools');
     nonExistingFunction('boom');
     ```
   - В DevTools → Console вы увидите красную ошибку `nonExistingFunction is not defined`.
   - Можно использовать для объяснения, как ошибки JS ломают функционал и почему важно проверять консоль.

2. **Конфликтные meta‑теги для SEO**
   - В `<head>` заданы:
     ```html
     <meta name="robots" content="noindex,nofollow" />
     <link rel="canonical" href="https://example.com/wrong-page" />
     ```
   - Через DevTools → Elements или исходник (`Ctrl+U`) можно показать:
     - как meta `robots` может полностью выключить индексацию страницы;
     - как неправильный `canonical` отправляет поисковик на другую страницу.

3. **Лишние скрипты и внешние ресурсы**
   - Подключён фейковый внешний скрипт:
     ```html
     <script src="https://cdn.example.com/heavy-analytics.js"></script>
     ```
   - Во вкладке Network видно дополнительный запрос и его параметры.
   - Это удобно для демонстрации того, как искать лишние пиксели/аналитику и оценивать «тяжесть» фронтенда.

4. **Мини‑код для проверки meta и canonical через Console**
   - В тексте страницы есть пример:
     ```js
     const robots = document.querySelector('meta[name="robots"]');
     const canonical = document.querySelector('link[rel="canonical"]');

     console.log('robots:', robots && robots.content);
     console.log('canonical:', canonical && canonical.href);
     ```
   - Можно использовать в DevTools → Console, чтобы показать быстрый способ проверки важных тегов без сторонних сервисов.

## Как запустить

Проект не требует сборки и зависимостей — это обычный статический HTML.

### Вариант 1: открыть как файл

1. Склонируйте репозиторий или скачайте файл:
   ```bash
   git clone https://github.com/USERNAME/devtools-demo.git
   cd devtools-demo
   
2. Откройте `index.html` (или `devtools_demo.html`, если вы так назвали файл) двойным кликом в браузере.
3. Нажмите `F12`, чтобы открыть DevTools, и переходите по вкладкам Console / Elements / Network.

### Вариант 2: запустить через локальный сервер

Если хотите работать именно через `http://localhost`:

#### Python 3

```bash
cd devtools-demo
python -m http.server 8000
```

После этого откройте в браузере:

```
http://localhost:8000/index.html
```

(или другое имя файла, если вы его изменили).
